# Planejamento Operacional e Cronograma de Sprints

## 1. Organização do Trabalho e Cadência
O projeto será executado em **Sprints semanais** focadas na elicitação, entendimento e validação do escopo. A validação com o cliente será predominantemente **assíncrona** via Acordo de Feedback.

### Ritos da Equipe
* **Segundas-feiras (20h):** Reunião síncrona interna (Processamento de feedbacks, status das tarefas, impedimentos e planejamento da nova Sprint).
* **Terças-feiras (Manhã):** Envio assíncrono de artefatos e documentos para validação da cliente.

### Estratégia de Escopo
O início do projeto terá caráter exploratório e iterativo. Após as fases de descoberta, os requisitos fundamentais serão submetidos a um **Marco de Congelamento (Scope Freeze)** para garantir a estabilidade do desenvolvimento a longo prazo.
